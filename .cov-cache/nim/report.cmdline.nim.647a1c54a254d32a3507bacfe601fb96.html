46 ## Splits a `command line`:idx: into several components.
47 ##
48 ## **Note**: This proc is only occasionally useful, better use the
49 ## `parseopt module <parseopt.html>`_.
50 ##
51 ## On Windows, it uses the `following parsing rules
52 ## <https://msdn.microsoft.com/en-us/library/17w5ykft.aspx>`_:
53 ##
54 ## * Arguments are delimited by white space, which is either a space or a tab.
55 ## * The caret character (^) is not recognized as an escape character or
56 ## delimiter. The character is handled completely by the command-line parser
57 ## in the operating system before being passed to the argv array in the
58 ## program.
59 ## * A string surrounded by double quotation marks ("string") is interpreted
60 ## as a single argument, regardless of white space contained within. A
61 ## quoted string can be embedded in an argument.
62 ## * A double quotation mark preceded by a backslash (\") is interpreted as a
63 ## literal double quotation mark character (").
64 ## * Backslashes are interpreted literally, unless they immediately precede
65 ## a double quotation mark.
66 ## * If an even number of backslashes is followed by a double quotation mark,
67 ## one backslash is placed in the argv array for every pair of backslashes,
68 ## and the double quotation mark is interpreted as a string delimiter.
69 ## * If an odd number of backslashes is followed by a double quotation mark,
70 ## one backslash is placed in the argv array for every pair of backslashes,
71 ## and the double quotation mark is "escaped" by the remaining backslash,
72 ## causing a literal double quotation mark (") to be placed in argv.
73 ##
74 ## On Posix systems, it uses the following parsing rules:
75 ## Components are separated by whitespace unless the whitespace
76 ## occurs within ``"`` or ``'`` quotes.
77 ##
78 ## See also:
79 ## * `parseopt module <parseopt.html>`_
