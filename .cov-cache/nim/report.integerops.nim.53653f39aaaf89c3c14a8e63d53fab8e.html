67 template mulImplFallback(name, T, U, conv) {.dirty.} =
68 #
69 # This code has been inspired by Python's source code.
70 # The native int product x*y is either exactly right or *way* off, being
71 # just the last n bits of the true product, where n is the number of bits
72 # in an int (the delivered product is the true product plus i*2**n for
73 # some integer i).
74 #
75 # The native float64 product x*y is subject to three
76 # rounding errors: on a sizeof(int)==8 box, each cast to double can lose
77 # info, and even on a sizeof(int)==4 box, the multiplication can lose info.
78 # But, unlike the native int product, it's not in *range* trouble: even
79 # if sizeof(int)==32 (256-bit ints), the product easily fits in the
80 # dynamic range of a float64. So the leading 50 (or so) bits of the float64
81 # product are correct.
82 #
83 # We check these two ways against each other, and declare victory if
84 # they're approximately the same. Else, because the native int product is
85 # the only one that can lose catastrophic amounts of information, it's the
86 # native int product that must have overflowed.
87 #
