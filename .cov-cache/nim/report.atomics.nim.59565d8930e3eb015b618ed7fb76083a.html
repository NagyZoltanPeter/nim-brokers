62 MemoryOrder* {.importcpp: "std::memory_order".} = enum
63 ## Specifies how non-atomic operations can be reordered around atomic
64 ## operations.
65
66 moRelaxed
67 ## No ordering constraints. Only the atomicity and ordering against
68 ## other atomic operations is guaranteed.
69
70 moConsume
71 ## This ordering is currently discouraged as it's semantics are
72 ## being revised. Acquire operations should be preferred.
73
74 moAcquire
75 ## When applied to a load operation, no reads or writes in the
76 ## current thread can be reordered before this operation.
77
78 moRelease
79 ## When applied to a store operation, no reads or writes in the
80 ## current thread can be reorderd after this operation.
81
82 moAcquireRelease
83 ## When applied to a read-modify-write operation, this behaves like
84 ## both an acquire and a release operation.
85
86 moSequentiallyConsistent
87 ## Behaves like Acquire when applied to load, like Release when
88 ## applied to a store and like AcquireRelease when applied to a
89 ## read-modify-write operation.
90 ## Also guarantees that all threads observe the same total ordering
91 ## with other moSequentiallyConsistent operations.
92
