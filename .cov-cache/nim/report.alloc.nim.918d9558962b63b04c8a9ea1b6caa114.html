19 # We manage *chunks* of memory. Each chunk is a multiple of the page size.
20 # Each chunk starts at an address that is divisible by the page size.
21 # Small chunks may be divided into smaller cells of reusable pointers to reduce the number of page allocations.
22
23 # An allocation of a small pointer looks approximately like this
24 #[
25
26 alloc -> rawAlloc -> No free chunk available > Request a new page from tslf -> result = chunk.data -------------+
27 | |
28 v |
29 Free chunk available |
30 | |
31 v v
32 Fetch shared cells -> No free cells available -> Advance acc -> result = chunk.data + chunk.acc -------> return
33 (may not add new cells) ^
34 | |
35 v |
36 Free cells available -> result = chunk.freeList -> Advance chunk.freeList -----------------------------------+
37 ]#
38 # so it is split into 3 paths, where the last path is preferred to prevent unnecessary allocations.
39 #
40 #
41 # A deallocation of a small pointer then looks like this
42 #[
43 dealloc -> rawDealloc -> chunk.owner == addr(a) --------------> This thread owns the chunk ------> The current chunk is active -> Chunk is completely unused -----> Chunk references no foreign cells
44 | | (Add cell into the current chunk) | Return the current chunk back to tlsf
45 | | | |
46 v v v v
47 A different thread owns this chunk. The current chunk is not active. chunk.free was < size Chunk references foreign cells, noop
48 Add the cell to a.sharedFreeLists Add the cell into the active chunk Activate the chunk (end)
49 (end) (end) (end)
50 ]#
