1 ## Multi-Thread Broker Queue Primitives
2 ## ------------------------------------
3 ## Lock-free MPSC primitives used to replace `Channel[T]` in the (mt)
4 ## brokers. Implements `doc/REFACTOR_MT_QUEUE.md` §3.
5 ##
6 ## Invariants enforced by *structural design*, not by runtime asserts:
7 ##
8 ## I0 every `createShared` / `deallocShared` runs on a persistent owner
9 ## thread (bucket-owner for per-bucket structures, global-slab-owner
10 ## for events). Hot path (claim / release / enqueue / dequeue) never
11 ## calls any Nim allocator.
12 ## CARVE-OUT (flexible-mt-dispatch): when a marshaled payload exceeds the
13 ## fixed cell, the producer `allocShared0`s a heap-spill buffer and the
14 ## consumer-side `release` `deallocShared`s it. So the spill path DOES
15 ## allocate on the hot path — a deliberate trade so oversized payloads
16 ## (>cell, e.g. >1 MiB) succeed instead of being dropped. The common
17 ## fits-the-cell path is unchanged and allocator-free. Spill buffers are
18 ## POD bytes with single producer→consumer ownership, same cross-thread
19 ## contract as `storage` itself.
20 ## I1 Senders only execute: atomic load / store / CAS, memcpy into
21 ## pre-allocated cells, and `ThreadSignalPtr.fireSync()` (external).
22 ## I2 The owner thread must outlive every structure it owns.
23 ##
24 ## Phase 1 deliverable: this module + its tests. No broker code calls it
25 ## yet — that lands in Phases 2-4.

134 # ---------------------------------------------------------------------------
135 #
136 # Atomic protocol:
137 # Producer (`tryEnqueue`):
138 # 1. closed-check (acquire) → if closed, return false.
139 # 2. pos = enqPos.load(relaxed)

143 # diff = seq - pos (as signed)

145 # CAS enqPos: pos → pos+1 (acquireRelease on success, acquire on failure)
146 # if success: break (slot is claimed; must publish)
147 # else: pos was reloaded into `expected`; loop
148 # elif diff < 0:
149 # return false (full)
150 # else:
151 # another producer claimed this slot already; reload pos, loop
152 # 4. write slot.payload
153 # 5. slot.seq.store(pos+1, release) -- publish
154 #
155 # Consumer (`tryDequeue`, single-thread):
156 # 1. pos = deqPos
157 # 2. seq = slots[pos & mask].seq.load(acquire)
158 # 3. if seq != pos+1: return false (empty / not yet published)
159 # 4. read payload
160 # 5. slots[pos & mask].seq.store(pos + capacity, release) -- slot reusable
161 # 6. deqPos = pos + 1

164 # 1. closed.store(true, release)

413 # ResponseSlot[T] + ResponseSlotPool[T] — single-shot request reply
414 # ---------------------------------------------------------------------------
415 #
416 # State machine on the slot's `state` byte:
417 # Empty(0) ── requester claimed; provider hasn't written yet
418 # │
419 # ├── (provider) CAS Empty→Ready, write payload, signal requester
420 # │ │
421 # │ └── (requester) read payload; release slot
422 # │
423 # └── (requester timeout) CAS Empty→Abandoned
424 # │
425 # └── (provider) sees Abandoned; releases slot
426 #
427 # In both terminal cases the slot returns to the pool's free-list
428 # exactly once.
429
