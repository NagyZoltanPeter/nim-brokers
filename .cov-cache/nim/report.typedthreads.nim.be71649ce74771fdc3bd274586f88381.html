10 ##[
11 Thread support for Nim. Threads allow multiple functions to execute concurrently.
12
13 In Nim, threads are a low-level construct and using a library like `malebolgia`, `taskpools` or `weave` is recommended.
14
15 When creating a thread, you can pass arguments to it. As Nim's garbage collector does not use atomic references, sharing
16 `ref` and other variables managed by the garbage collector between threads is not supported.
17 Use global variables to do so, or pointers.
18
19 Memory allocated using [`sharedAlloc`](./system.html#allocShared.t%2CNatural) can be used and shared between threads.
20
21 To communicate between threads, consider using [channels](./system.html#Channel)
22
23 Examples
24 ========
25
26 ```Nim
27 import std/locks
28
29 var
30 thr: array[0..4, Thread[tuple[a,b: int]]]
31 L: Lock
32
33 proc threadFunc(interval: tuple[a,b: int]) {.thread.} =
34 for i in interval.a..interval.b:
35 acquire(L) # lock stdout
36 echo i
37 release(L)
38
39 initLock(L)
40
41 for i in 0..high(thr):
42 createThread(thr[i], threadFunc, (i*10, i*10+5))
43 joinThreads(thr)
44
45 deinitLock(L)
46 ```
47
48 When using a memory management strategy that supports shared heaps like `arc` or `boehm`,
49 you can pass pointer to threads and share memory between them, but the memory must outlive the thread.
50 The default memory management strategy, `orc`, supports this.
51 The example below is **not valid** for memory management strategies that use local heaps like `refc`!
52
53 ```Nim
54 import locks
55
56 var l: Lock
57
58 proc threadFunc(obj: ptr seq[int]) {.thread.} =
59 withLock l:
60 for i in 0..<100:
61 obj[].add(obj[].len * obj[].len)
62
63 proc threadHandler() =
64 var thr: array[0..4, Thread[ptr seq[int]]]
65 var s = newSeq[int]()
66
67 for i in 0..high(thr):
68 createThread(thr[i], threadFunc, s.addr)
69 joinThreads(thr)
70 echo s
71
72 initLock(l)
73 threadHandler()
74 deinitLock(l)
75 ```
