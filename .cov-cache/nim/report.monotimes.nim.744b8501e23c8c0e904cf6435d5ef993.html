10 ##[
11 The `std/monotimes` module implements monotonic timestamps. A monotonic
12 timestamp represents the time that has passed since some system defined
13 point in time. The monotonic timestamps are guaranteed not to decrease,
14 meaning that that the following is guaranteed to work:
15 ]##
16
17 runnableExamples:
18 let a = getMonoTime()
19 let b = getMonoTime()
20 assert a <= b
21
22 ##[
23 This is not guaranteed for the `times.Time` type! This means that the
24 `MonoTime` should be used when measuring durations of time with
25 high precision.
26
27 However, since `MonoTime` represents the time that has passed since some
28 unknown time origin, it cannot be converted to a human readable timestamp.
29 If this is required, the `times.Time` type should be used instead.
30
31 The `MonoTime` type stores the timestamp in nanosecond resolution, but note
32 that the actual supported time resolution differs for different systems.
33
