1 ## MultiRequestBroker
2 ## --------------------
3 ## MultiRequestBroker represents a proactive decoupling pattern, that
4 ## allows defining request-response style interactions between modules without
5 ## need for direct dependencies in between.
6 ## Worth considering using it for use cases where you need to collect data from multiple providers.
7 ##
8 ## Generates a standalone, type-safe request broker for the declared type.
9 ## The macro exports the value type itself plus a broker companion that manages
10 ## providers via thread-local storage.
11 ##
12 ## Unlike `RequestBroker`, every call to `request` fan-outs to every registered
13 ## provider and returns all collected responses.
14 ## The request succeeds only if all providers succeed, otherwise it fails.
15 ##
16 ## Type definitions:
17 ## - Inline `object` / `ref object` definitions are supported.
18 ## - Native types, aliases, and externally-defined types are also supported.
19 ## In that case, MultiRequestBroker will automatically wrap the declared RHS
20 ## type in `distinct` unless you already used `distinct`.
21 ## This keeps request types unique even when multiple brokers share the same
22 ## underlying base type.
23 ##
24 ## Default vs. context aware use:
25 ## Every generated broker is a thread-local global instance.
26 ## Sometimes you want multiple independent provider sets for the same request
27 ## type within the same thread (e.g. multiple components). For that, you can use
28 ## context-aware MultiRequestBroker.
29 ##
30 ## Context awareness is supported through the `BrokerContext` argument for
31 ## `setProvider`, `request`, `removeProvider`, and `clearProviders`.
32 ## Provider stores are kept separate per broker context.
33 ##
34 ## Default broker context is defined as `DefaultBrokerContext`. If you don't
35 ## need context awareness, you can keep using the interfaces without the context
36 ## argument, which operate on `DefaultBrokerContext`.
37 ##
38 ## Usage:
39 ##
40 ## Declare collectable request data type inside a `MultiRequestBroker` macro, add any number of fields:
41 ## ```nim
42 ## MultiRequestBroker:
43 ## type TypeName = object
44 ## field1*: Type1
45 ## field2*: Type2
46 ##
47 ## ## Define the request and provider signature, that is enforced at compile time.
48 ## proc signature*(): Future[Result[TypeName, string]] {.async: (raises: []).}
49 ##
50 ## ## Also possible to define signature with arbitrary input arguments.
51 ## proc signature*(arg1: ArgType, arg2: AnotherArgType): Future[Result[TypeName, string]] {.async: (raises: []).}
52 ##
53 ## ```
54 ##
55 ## You can register a request processor (provider) anywhere without the need to
56 ## know who will request.
57 ## Register provider functions with `TypeName.setProvider(...)`.
58 ## Providers are async procs or lambdas that return `Future[Result[TypeName, string]]`.
59 ## `setProvider` returns a handle (or an error) that can later be used to remove
60 ## the provider.
61
62 ## Requests can be made from anywhere with no direct dependency on the provider(s)
63 ## by calling `TypeName.request()` (with arguments respecting the declared signature).
64 ## This will asynchronously call all registered providers and return the collected
65 ## responses as `Future[Result[seq[TypeName], string]]`.
66 ##
67 ## Whenever you don't want to process requests anymore (or your object instance that provides the request goes out of scope),
68 ## you can remove it from the broker with `TypeName.removeProvider(handle)`.
69 ## Alternatively, you can remove all registered providers through `TypeName.clearProviders()`.
70 ##
71 ## Example:
72 ## ```nim
73 ## MultiRequestBroker:
74 ## type Greeting = object
75 ## text*: string
76 ##
77 ## ## Define the request and provider signature, that is enforced at compile time.
78 ## proc signature*(): Future[Result[Greeting, string]] {.async: (raises: []).}
79 ##
80 ## ## Also possible to define signature with arbitrary input arguments.
81 ## proc signature*(lang: string): Future[Result[Greeting, string]] {.async: (raises: []).}
82 ##
83 ## ...
84 ## let handle = Greeting.setProvider(
85 ## proc(): Future[Result[Greeting, string]] {.async: (raises: []).} =
86 ## ok(Greeting(text: "hello"))
87 ## )
88 ##
89 ## let anotherHandle = Greeting.setProvider(
90 ## proc(): Future[Result[Greeting, string]] {.async: (raises: []).} =
91 ## ok(Greeting(text: "szia"))
92 ## )
93 ##
94 ## let responses = (await Greeting.request()).valueOr(@[Greeting(text: "default")])
95 ##
96 ## echo responses.len
97 ## Greeting.clearProviders()
98 ## ```
