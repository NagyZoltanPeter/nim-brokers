# C++ Consumer

Modern C++ text UI for the Torpedo Duel example.

Implemented in [main.cpp](main.cpp).

The C++ app mirrors the Python consumer — it creates two independent
`torpedolib` contexts, bootstraps the duel, then observes it through event
callbacks and board polling while the captains fight autonomously inside Nim.

Compared to the Python version, it adds ANSI color output: ships in green,
hits in yellow, sunk segments in red, and dimmed misses.

## Build And Run

From the repository root:

```text
nimble buildTorpedoExampleCpp
nimble runTorpedoExampleCpp
```

For a faster run during development:

```text
nimble runTorpedoExampleCpp -- --fast
```

Choose the opening player:

```text
nimble runTorpedoExampleCpp -- --starter blue
```

## Requirements

- CMake 3.16+
- C++20 compiler (GCC 10+, Clang 13+, MSVC 2019+)
- The Nim shared library must be built first (`nimble buildTorpedoExample`)

## Architecture

The C++ wrapper class (`Torpedolib`) is generated by the `nim-brokers` FFI
API layer and lives in `torpedolib.h`. It provides:

- RAII lifecycle (`createContext()` / `shutdown()` in destructor)
- `Result<T>` return types for all requests
- `std::string_view` in event callbacks (zero-copy, valid during call)
- `EventDispatcher` template for thread-safe callback multiplexing
- Handle-based event listener management (`onXxx()` / `offXxx()`)

See [../DESIGN.md](../DESIGN.md) for the full interaction model.
